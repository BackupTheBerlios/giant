\section{Über die Benutzeroberfläche}

\begin{enumerate}
  \item {GIANT besitzt eine graphische Benutzeroberfläche, die per Maus zu
  bedienen ist. Ausgewählte Funktionen können auch per Tastatur ausgelöst
  werden.}
  \item {Die Interaktionssprache mit dem Benutzer ist Englisch.}
\end{enumerate}


\section{Über dieses Kapitel}    

Hier werden alle Elemente der GUI beschrieben. Sofern nicht anders
angegeben, sind hintereinander angegebene Elemente (z.B. Listeneinträge)
immer von links nach rechts oder von oben nach unten beschrieben.
Alle Ausgaben sind immer linksbündig formatiert, sofern nicht anders
vermerkt.


\section {Main Window}

\begin{figure}[h!]
   \centering
   \includegraphics[scale=.25]{Main_Window}
   \caption{Main-Window}
\end{figure}

Jede Instanz von GIANT hat genau ein Hauptfenster.
Im Hauptfenster (MAIN\_WINDOW) werden vorhandene Anzeigefenster und
IML-Teilgraphen angezeigt. Über Popup-Menüs können IML-Teilgraphen
und Anzeigefenster manipuliert werden.

Im Fenster befinden sich zwei Listen, WINDOW\_LIST (2 Spalten), und
SUBGRAPH\_LIST (4 Spalten).


  \subsection {Kopfzeile}
  In der Kopfzeile wird \gq{GIANT - <Projektname>} dargestellt.
 
  \subsection {Menüleiste (Main Window)}
  In der Menüleiste befinden sich folgende Einträge:

    \subsubsection {Untermenü File } \label{Main-Window-File}
      \begin{enumerate}
         \item {New Project} % OK
         \item {Load Project}
         \item {Save Project}
         \item {Save Project As...}
         \item {Quit}
      \end{enumerate}    
      
     \subsubsection {Untermenü Tools}  \label{Main-Window-Tools}
       \begin{enumerate}
          \item {Delete Annotations Having No Visible Node}
          \item {Execute GQSL Query}
        \end{enumerate}
        

    \subsubsection {Untermenü Info }  \label{Main-Window-Info}     
       \begin{enumerate}
          \item {About GIANT}
        \end{enumerate}
     

  \subsection {Statuszeile}
  \label{Statuszeile}
   Die Statuszeile befindet sich ganz unten im Fenster.
   
   Gegebenenfalls werden in der Statustzeile Informationen zum aktuellen
   Zustand von GIANT dargestellt.

   \subsection {Window List}
  
   Im Fenster befindet sich zuoberst eine Liste WINDOW\_LIST, welche
   alle Fenster des Projektes (offen und geschlossen anzeigt).
   
   \subsubsection {Inhalt Window List}
   \label{WINDOW-LIST}
   
   WINDOW\_LIST hat die Spalten WINDOW\_NAME \gq{Visualisation Windows} und WINDOW\_OPEN \gq{Open}. In
   WINDOW\_NAME wird der Name aller Anzeigefenster dargestellt, in
   WINDOW\_OPEN befindet sich entweder ein Strich-Symbol (Fenster
   nicht geöffnet) oder ein Häkchensymbol (Fenster geöffnet).
    
    \subsubsection {Popup-Menü Window List}
    
    
        Beim Rechtsklick auf WINDOW\_LIST öffnet sich ein 
        Popup-Menü mit folgenden Menüpunkten:
        \label{WINDOW-LIST-POPUP}

        \begin{enumerate}
          \item Open Window % OK
          \item Close Window % OK
          \item Delete Window % OK
          \item New Window % OK
        \end{enumerate}
    
  \subsection {Subgraph List}
  \label{GUI Subgraph List}
    
    Unter der WINDOW\_LIST im Fenster befindet sich eine Liste SUBGRAPH\_LIST.
   
    \subsubsection {Inhalt Subgraph List} 
    
      SUBGRAPH\_LIST hat die Spalten 
      \begin{enumerate}
        \item Subgraph Name (SUBGRAPH\_NAME)
        \item Nodes (SUBGRAPH\_NODES)
        \item Edges (SUBGRAPH\_EDGES)
        \item Highlight Color (SUBGRAPH\_HIGHLIGHT\_COL).
      \end{enumerate}
    
      In Subgraph Name stehen die Namen der existierenden Subgraphen,
      in Nodes die Anzahl der Knoten des Subgraphen, in Edges die
      Anzahl der Kanten und in Highlight Color ein Kästchen mit der
      Farbe, die verwendet wird, wenn dieser Subgraph markiert wird.

    
    \subsubsection {Popup-Menü Subgraph List}
    \label {Popup-Menü Subgraph List}
      Beim Rechtsklick auf SUBGRAPH\_LIST öffnet 
      sich ein Popup-Menü mit folgenden Menüpunkten:
        \label{SUBGRAPH-LIST-POPUP}
        \begin{enumerate}
          \item Highlight  Color 1 / Color 2 / Color 3 (drei Untermenüs)
          \item Unhighlight In All Windows
          \item Copy IML Subgraph
          \item Create Window Selection % OK
          \item Delete IML Subgraph
          \item IML Subgraph Set Operation 
        \end{enumerate}   
        

\section {Anzeigefenster} \label{GUI Anzeigefenster}
\index{Anzeigefenster}

  \begin{figure}[!htbp]
     \centering
     \includegraphics[scale=.5]{Graph_Window}
     \caption{Graph-Window}
  \end{figure}

  Im Programm kann es mehrere Anzeigefenster geben. Ein Anzeigefenster besteht
  aus folgenden Elementen:

  In Anzeigefenstern (VISUALIZATION\_WINDOW) werden Graphen visualisiert.
  Sie erscheinen im großen quadratischen rechten Teil (VIS\_PANE) 
  des Fensters. Die VIS\_PANE enthält den Anzeigeinhalt des Fensters.
  Im linken Viertel des Fensters, der Toolbar (VISUALIZATION\_WINDOW\_TOOLBAR)
  befinden sich untereinander die MiniMap (MINIMAP), Pinliste (PIN\_LIST)
  mit darunterliegenden Buttons, Selektionsliste (SELECTION\_LIST),
  die Zoom-Kontrolle, 
  die Stilauswahl-Combobox (STYLE\_COMBO)
  \label{GUI Stilauswahl-Combobox} 
  und  ein Button \gq{Hide} zum Ausblenden der Toolbar (TOOLBAR\_HIDE).

 \subsection {MiniMap} \label{GUI Minimap}

    In der MiniMap wird der gesamte IML-Teilgraph, der im Fenster
    visualisiert ist, durch einen grauen Kasten repräsentiert.
    Dabei repräsentiert der graue Kasten die Ausdehnung des gesamten
    Graphen inklusive der z.Z.\ nicht im Fenster sichtbaren Bereiche.
    Der im Fenster sichtbare Bereich wird in der MiniMap durch einen
    Rahmen angezeigt. Je nach Zoomstufe ist der sichtbare Bereich
    größer oder kleiner.
    Durch Klicks in die Minimap kann die linke obere Ecke des
    sichtbaren Bereiches versetzt werden, der im Anzeigeinhalt VIS\_PANE angezeigte
    Bereich wird dann entsprechend angepasst.

  \subsection{Visualisierung der Knoten}

    In der VIS\_PANE werden die Fenster-Knoten und Fenster-Kanten angezeigt.
    
    Wenn auf einem Knoten die rechte Maustaste geklickt wird, öffnet sich
    folgendes Popup-Menü:
 
    \subsubsection{Node-Popup-Menü}\label{Node-Popup-Menü}
    \begin{enumerate}
      \item Show Node Info Window
      \item Show Corresponding Source Code
      \item Annotation Add / Change / Delete (drei Untermenüs)
      \label{Change Node Annotation}
      \label{Delete Node Annotation}
      \item Show Node Annotation \label{Show Node Annotation}
    \end{enumerate}
    
    Wenn in der VIS\_PANE mit der rechten Maustaste auf eine Stelle
    geklickt wird an der kein Knoten liegt, öffnet sich das folgende
    Popup-Menü. Die Einträge dieses Menüs sind auch im Node-Popup-Menü
    (siehe \ref{Node-Popup-Menü}) sichtbar.

    \label{Empty Vis Pane Right click}
    \begin{enumerate}
      \item Make Room % OK
      \item New Pin % OK
    \end{enumerate}
    

  \subsection{Pins} \label{VIS-PANE-Pins}

    In der Pinliste (PIN\_LIST) können Pins festgelegt werden.

    Im Popup-Menü der Pinliste befinden sich folgende Menüpunkte:
    
    \begin{enumerate}
      \item Focus Pin (PIN\_JUMP) % OK
      \item Delete Pin (PIN\_DELETE) % OK
    \end{enumerate}

  \subsection{Selektionsauswahlliste} \label{Selektionsauswahlliste}
    Die Selektionsauswahlliste hat die Spalten Selection (mit dem Namen der
    Selektion) und State (mit dem Status der Selektion, also z.B. farbig
    hervorgehoben oder nicht).

    Im Popup-Menü der Selektionsauswahlliste (SELECTION\_LIST)
    befinden sich folgende Einträge:

    \begin{enumerate}
      \item New Selection

      \item Move Selection
     
      \item Show Selection % war: Fade In Selection
      \item Hide Selection % war: Fade Out Selection

      \item Layout Selection

      \item Create New IML Subgraph From This Selection % 
      
      \item Highlight Selection Color1 / Color 2 / Color 3 (drei Untermenüs) % OK
      
      \item Zoom To Make Selection Fill Window % OK
     
      \item Copy Selection
      
      \item Unhighlight Selection % OK
      \item Copy Selection Keep Existing Layout % OK
      \item Copy Selection Change Existing Layout % OK
      \item Delete Selection % OK
      \item Selection Set Operation (Union/Difference/Intersection) % OK
      
      \item Set Layout Algorithm For Selection
      
    \end{enumerate}    


  \subsection{Zoom-Kontrolle} \label{GUI Zoom-Kontrolle}  

    Die Zoom-Kontrolle besteht aus folgenden Elementen:
    
  \begin{enumerate}
      \item Combobox ZOOM\_CONTROL mit vorgefertigten Werten und Eingabemöglichkeit % OK
      \item Button \gq{Display} (ZOOM\_DISPLAY) % OK
      \item Button \gq{Zoom+} (ZOOM\_PLUS) % OK
      \item Button \gq{Zoom-} (ZOOM\_MINUS) % OK
      \item Button \gq{Fill Window} (ZOOM\_FILL) % OK
      \item Button \gq{Zoom Edge} (ZOOM\_EDGE) % OK
  \end{enumerate}    
     
  \subsection{Stilauswahl-Combobox}

    In der Stilauswahl-Combobox (STYLE\_CHOOSER) kann ein Anzeigestil
    festgelegt werden.

  \subsection{Ein- und Ausklappbutton}

    Durch Klick auf den Ein- und Ausklappbutton (VIS\_PANE\_BUTTON) kann die VIS\_PANE
    ein- und ausgeblendet werden.
    
    \subsection{Bildlaufleisten}
    \label{Scrolleisten}
    
    An den beiden Bildlaufleisten (links/rechts und oben/unten) in
    jedem Anzeigefenster kann der im Fenster sichtbare Bereich des
    Graphen verschoben werden.

\section{Knoten-Informationsfenster} \label{Knoten-Informationsfenster}

    \begin{figure}[!htbp]
       \centering
       \includegraphics{Node_Info_Window}
       \caption{Node-Info-Window}
    \end{figure}

    In Knoten-Informationsfenstern (NODE\_INFO) können nähere Informationen
    zu Knoten angezeigt werden.

    Im Fenster wird die Knotentyp und Knoten-ID dargestellt
    (TYPE und ID), Format: "Node: TYPE, ID: ID), darunter befinden sich
    (jeweils untereinander) drei getrennt scrollbare Listen,
    ATTRIBUTES, SUCCESSOR\_NODES und PREDECESSOR\_NODES.
    
    Unter den drei Listen befinden sich nebeneinander mittig zwei Buttons
    CLOSE und PICK (zum Auswählen eines anderen Knotens zur Anzeige).
    
    \subsection{Liste ATTRIBUTES}

      Die Liste ATTRIBUTES enthält alle Attribute des Knotens und hat zwei Spalten:
    
      \begin{enumerate}
        \item ATTRIBUTE (Attribute Name)
        \item VALUE (Attribute Value)
      \end{enumerate}
    
    \subsection{Liste SUCCESSOR\_NODES}
    
      Die Liste SUCCESSOR\_EDGES hat eine Zeile für jede vom Knoten abgehende
      Kante mit zwei Spalten:
    
    \begin{enumerate}
      \item EDGE\_TYPE (Typ der abgehenden Kante: Edge Type)
      \item NODE\_ID (ID des Knotens, zu dem sie führt: Node ID)
    \end{enumerate}    

    \subsection{Liste PREDECESSOR\_NODES}
      
      Die Liste PREDECESSOR\_EDGES hat eine Zeile für jede vom Knoten abgehende
      Kante mit zwei Spalten:

      \begin{enumerate}
        \item EDGE\_TYPE (Typ der ankommenden Kante: Edge Type)
        \item NODE\_ID (ID des Knotens, von dem sie kommt: Node ID)
      \end{enumerate}

\section{Common\_Set\_Operation\_Dialog}
\label{Common-Set-Operation-Dialog}

    \begin{figure}[!htbp]
       \centering
       \includegraphics[scale=.8]{Common_Set_Operation_Dialog}
       \caption{Common-Set-Operation-Dialog}
    \end{figure}


Dieser Dialog dient dazu, Mengenoperationen über Selektionen und
IML-Teilgraphen durchzuführen.\\
Die Mengenoperation lässt sich dann wie folgt beschreiben:
LEFT\_SOURCE <op> RIGHT\_SOURCE := TARGET \\ \\


Bestandteile des Dialoges sind:
\begin {enumerate}

 \item LEFT\_SOURCE Combobox\\
 Soll eine Mengenoperation für Selektionen ausgeführt werden, so können hier
 alle Selektionen des entsprechenden Anzeigefensters ausgewählt werden.\\
 Bei einer Mengenoperation über IML-Teilgraphen, werden alle IML-Teilgraphen
 des Projektes angezeigt.
 
 \item <OP> Combobox\\
 Mengenoperation, die ausgeführt werden soll: Union, Difference oder Intersection.

 \item RIGHT\_SOURCE Combobox\\
 Soll eine Mengenoperation für Selektionen ausgeführt werden, so können hier
 alle Selektionen des entsprechenden Anzeigefensters ausgewählt werden.\\
 Bei einer Mengenoperation über IML-Teilgraphen, werden alle IML-Teilgraphen
 des Projektes angezeigt.
 
 \item TARGET Textfield\\
 Name der neuen Selektion oder des neuen IML-Teilgraphen als Ergebnis
 der Mengenoperation.
 
\end {enumerate}

\section{Fadenkreuz-Cursor}
\label{Fadenkreuzcursor}
        Der Cursor verwandelt sich nach Auswahl bestimmter
        Funktionen in ein Fadenkreuz.
        Durch Linksklick auf einen Knoten, eine Kante oder leere Position in einem
	Fenster wird diese(r) für die jeweilige Funktion ausgewählt. Der Cursor verwandelt
        sich dann wieder in den Standard-Cursor.
	Durch Rechtsklick läßt sich die ausgewählte Funktion abbrechen.


\section{Fortschrittsanzeige}

\label{Progressbar}
Es gibt in GIANT zwei Möglichkeiten zur Fortschrittsanzeige während laufender Berechnungen.
Jede Progressbar ist mit \gq{Working, please wait.} beschriftet.

\subsection{Progressbar-Runs}
\label{Progressbar-Runs}
     Fenster mit Anzeige eines Dialoges mit einem sich ständig leicht ändernden
     Inhalt, so dass erkennbar ist, dass das System noch arbeitet.
     \begin{figure}[!htbp]
      \centering
      \includegraphics{Progressbar-Runs}
      \caption{Progressbar-Runs}
     \end{figure}

\subsection{Progressbar-Modal}
\label{Progressbar-Modale} 
     Fenster mit Anzeige eines Progressbars, der über den aktuellen Fortschritt
     einer Berechnung informiert, ohne dabei den Gesamtaufwand zu 
     kennen. Dabei wird kontinuierlich die Anzahl der schon bearbeiteten
     Datensätze ausgegeben, und, sofern bekannt, die Prozentzahl der schon
     bearbeiteten Datensätze als Text.
     Während dieser Dialog sichtbar ist, ist der Rest der GUI von GIANT
     gesperrt.
     Im Fenster existiert ein Button \gq{Cancel} \label{Progressbar-Modale-Cancel}, der die momentane
     Berechnung abbricht.
     \begin{figure}[!htbp]
      \centering
      \includegraphics{Modal-Progressbar}
      \caption{Progressbar-Modal}
     \end{figure}
