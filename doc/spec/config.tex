\section {Allgemeines}

Sämtliche konfigurierbaren Einstellungen werden in XML-Dateien vorgenommen.
Es gibt genau eine\gq{globale Konfigurationsdatei}. Zudem kann es beliebig viele weitere 
XML-Dateien zur Konfiguration der weiter unten beschriebenen Visualisierungsstiele
geben.

\section {Die globale Konfigurationsdatei}
Diese Datei ist in XML verfasst und existiert genau ein mal. In ihr werden 
die anschließend beschriebenen Einstellungen vorgenommen.

  \subsection {Verweise auf Menü-Makros}
  Es können beliebig viele Menü-Makros definiert werden.
  Jeder Makro wird durch einen eindeutigen Namen und durch einen Verweis
  auf die Datei, welche den Makro enthält, spezifiziert.

  \subsection {Weitere Einstellungen}
  MÜSSEN WIR UNS NOCH AUSDENKEN.

  \subsection {Verweise auf Visualisierungsstiele}
  EVENTUELL -> KLÄREN OB DIE XML DATEI, DIE EINEN VISUALISIERUNGSTIEL ENTHÄLT HIER REFERENZIERT WERDEN
  MUSS ODER OB MAN NICH PAUSCHAL ALLE XML-DATEIEN MIT VISUALISIERUNGSSTIELEN IN EIN 
  VORGEGEBENES VERZEICHNIS KOPIEREN SOLL.

\section {Visualisierungsstiele}
Für jeden Visualisierungsstiel muss es eine entsprechende XML-Datei geben. Ein Visualisierungsstiel 
beschreibt wie die Knoten und Kanten des IML-Graphen innerhalb eines Anzeigefensters grafisch dargestellt 
werden können.\\
In einem Visualisierungsstiel können klassenspezifische Einstellungen vorgenommen werden,
die nur für Knoten und Kanten gelten, die zu den entsprechenden Klassen gehören. Bei jeder 
klassenspezifischen Einstellung für Knoten und Kanten kann daher eine Liste der betroffenen
Knoten- und Kantenklassen angegeben werden, für die diese Einstellungen gelten sollen.\\
Wird eine Kanten- oder eine Knotenklasse innerhalb eines Visualisierungsstieles mehreren 
klassenspezifischen Einstellungen zugeordnet, führt dies zu keinem Fehler, es bleibt aber
unspezifiziert, welche Einstellung tatsächlich genommen wird.

  \subsection {Name des Visualisierungsstiels}
  Jeder Visualisierungsstiel erhält einen Namen. Unter diesem Namen ist der 
  Visualisierungsstiel in der GUI (bei den Anzeigefenstern) auswählbar.
  Der Name muss nicht eindeutig sein oder dem Namen der Datei, 
  in der der Visualisierungssteil gespeichert ist, entsprechen.


  \subsection {Einstellungen innerhalb des Visualisierungssteils}
    \begin{itemize}
      \item Die Hintergrundfarbe im Anzeigefenster.
      \item Die Position der Kantenbeschriftung. 
        \subitem Immer im sichtbaren Anzeigeinhalt.
        \subitem Pauschal in der Mitte der Kante. 
        \subitem Am Startknoten.
        \subitem Am Zielknoten.
      \item Wann Kanten im sichtbaren Anzeigeinhalt angezeigt werden sollen.
        ACHTUNG - ES KANN AUCH SCHLEIFEN GEBEN
        \subitem Nur falls Start- und Zielknoten ebenfalls im sichtbaren Anzeigeinhalt.
        \subitem Auch anzeigen falls nur Startknoten im sichtbaren Anzeigeinhalt.
        \subitem Auch anzeigen falls nur Zielknoten im sichtbaren Anzeigeinhalt.
        \subitem Auch anzeigen falls weder Start- und Zielknoten im sichtbaren Anzeigeinhalt. 
    \end{itemize}


  \subsection {Klassenspezifische Einstellungen für Knoten}
  Folgende  Einstellungen können für Knotenklassen vorgenommen werden.
  Des weiteren muss eine DEFAULT-Einstellung erstellt werden, die für alle
  Knotenklassen angewendet wird, für die nichts anderes definiert ist.
  \begin{itemize}  
    \item Ein Icon für die Knotenklasse (Verweis auf eine entsprechende Bilddatei).  
    \item Eine Liste der Attribute der Knotenklasse, welche dirket innerhalb des
          Anzeigefensters in dem \gq(Viereck) für den Knoten dargestellt werden sollen.
    \item Die Füllfarbe des \gq{Vierecks} in welchem die Attribute zu dem Knoten dargestellt werden.
    \item Die Farbe des Rahmens für das \gq{Viereck} in welchem die Attribute zu dem Knoten dargestellt werden.
    \item EVENTUELL MUSS DIE MAXIMALE GRÖSSE (Breite und Länge) DES VIERECKS AUCH DEFINIERT WERDEN KÖNNEN
    \item Ob die ID eines Knotens im Anzeigefenster angezeigt werden soll oder nicht.
  \end{itemize}

  
  \subsection {Klassenspezifische Einstellungen für Kanten}
  Folgende Einstellungen können für Kantenklassen vorgenommen werden. 
  Des weiteren muss eine DEFAULT-Einstellung erstellt werden, die für alle
  Kantenklassen angewendet wird, für die nichts anderes definiert ist.
  \begin{itemize}
    \item Die Farbe der Kante.
    \item Die Art der Linie der Kante (normal, gestrichelt).
    \item Ob die Kante mit ihrer Kantenklasse beschriftet werden soll oder nicht.
  \end{itemize}
