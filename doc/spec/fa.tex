\section{Der Aktor \gq{Benutzer}} \index{Aktor} \index{Benutzer}
Aktor im Sinne der anschließend spezifizierten UseCases ist immer die 
Person, die den IML-Browser GIANT gerade bedient, also der 
\gq{Benutzer}. Neben dem \gq{Benutzer} sind keine weiteren Aktoren 
vorgesehen.\\


  \subsection{Anforderungen an den Aktor \gq{Benutzer}}
  Zur Bedienung von GIANT muss der Benutzer über 
  die folgenden Kenntnisse verfügen:

  \begin{enumerate}
    \item Erfahrung im Umgang mit grafischen Benutzeroberflächen und 
          dem entsprechenden Betriebssystem.
  
    \item Kenntnisse über Struktur und Aufbau des IML-Graphen. 
    
    \item Kenntnisse in der Anfragesprache GQSL von GIANT.

  \end{enumerate}

\section{Starten von GIANT} \index{Starten von GIANT}
\label{fa Starten von GIANT}

\index{Kommandozeile}
Der Benutzer startet das Programm GIANT durch Ausführen der entsprechenden
Programmdatei. Hierbei hat er bezüglich eventueller Kommandozeilenparameter
die folgenden Optionen. 

  \subsection {Mögliche Parameter} \index{Kommandozeilenparameter}
  \begin {enumerate}
    
    \item
    \verb+-loadiml=''Pfad''+; Abkürzung: \verb+-li=''Pfad''+ \\
    Angabe des Pfades zu einer existierenden IML-Graph Datei.
    
    \item 
    \verb+-newprojectfile=''Pfad''+; Abkürzung: \verb+-np''Pfad''+\\
    Angabe des Pfades für die neu zu erstellenden Projektdatei 
    (die Datei darf nicht bereits vorhanden sein, sonst Fehlermeldung).\\
    Das Verzeichnis der Projektdatei ist automatisch Projektverzeichnis,
    falls das Verzeichnis schon vorhanden ist, darf darin keine 
    weitere Projektdatei liegen.
    Existiert das Projektverzeichnis noch nicht, so wird es neu angelegt.
    
    \item 
    \verb+-loadproject=''Pfad''+; Abkürzung: \verb+-lp=''Pfad''+ \\
    Angabe des Pfades zu einer existierenden Projektdatei.
          
    \item
    \verb+-loadquery=''Pfad''+; Abkürzung \verb+-lq=''Pfad''+\\
    Pfad zu einer Datei in der ein GQSL Skript gespeichert ist.
  
    \item 
    \verb+-help+\\
    Zeigt Hilfe zu den möglichen Kommandozeilenparametern an.
        
      
  \end {enumerate}
      
  \subsection {Mögliche Aufrufe}

  Hier werden mögliche Parameterfolgen zum Start von GIANT via
  Kommandozeile erläutert.

  \begin {enumerate}
  
    \item Eingabe von: \verb+giant+\\
    GIANT startet und zeigt das Main Window an. 
    Es ist noch kein Projekt geladen.
    
    \item Eingabe von: \verb+giant -loadiml=''my\_iml\_file''+\\
    \verb+-newprojectfile=''/home/projektverzeichnis/projektdatei''+\\
    Ein neues Projekt wird geöffnet und erhält den Namen der Projektdatei.
    \label{Kommandozeile_neus}

    \item Eingabe von: \verb+giant -loadproject=''my\_project\_file''+\\
    Das angegebene Projekt wird geöffnet.
    \label{Kommandozeile_altes}
    
    \item
    Bei \ref{Kommandozeile_neus} und \ref{Kommandozeile_altes}
    kann zusätzlich entweder der Parameter\\
    \verb+... -loadquery''my\_query\_file''+ eingegeben werden+\\
    In diesem Falle wird dann die übergebene Anfrage ausgeführt.
    Die Ergebnisse werden dem neuen oder dem bestehenden Projekt 
    hinzugefügt, sind aber noch nicht persistent gespeichert.


  \end {enumerate}

\section{Beenden von GIANT} \index{Beenden von GIANT}
GIANT kann durch Auswahl des entsprechenden Eintrags im Hauptmenü 
beendet werden.\\
Ablauf:

\begin {enumerate}
  \item Der Benutzer gibt das Kommando zum Beenden von GIANT.
  \item 
  Ist ein Projekt geöffnet, so fragt GIANT nach, ob das Projekt gespeichert
  werden soll. Falls der Benutzer das Speichern ablehnt, so gehen alle
  nicht gespeicherten Informationen verloren.\\
  Entscheidet er sich für speichern, so wird 
  die unter \ref{Alles Speichern} beschriebene Funktionalität 
  ausgeführt.
 
    
  \item 
  Das System GIANT wird beendet.
    
\end {enumerate}
