\section{Der Aktor \gq{Benutzer}}
Aktor im Sinne der anschließend spezifizierten UseCases ist immmer die 
menschliche Person, die den IML-Browser GIANT gerade benutzt, also der 
\gq{Benutzer}. Da GIANT keinen Mehrbenutzerbetrieb vorsieht, ist dies immer 
eine einzige Person. 
Neben dem \gq{Benutzer} sind keine weiteren Aktoren vorgesehen.\\


  \subsection{Anforderungen an den Aktor \gq{Benutzer}}
  GIANT ist eine Profi-Werkzeug. Weder das System noch das Handbuch richten 
  sich an unerfahrene Benutzer. Zur Bedienung muss der Benutzer daher 
  zwingend über die folgenden Kenntnisse verfügen:

  \begin{itemize}
    \item Erfahrung im Umgang mit grafischen GUIs und 
          dem entsprechenden Betriebssystem.
  
    \item Grundkenntnisse in XML 
          (nur für das Editieren der Konfigurationsdateien).

    \item Erfahrung im Bereich Reengineering.

    \item Kenntnisse über Struktur und Aufbau des IML-Graphen. 
    
    \item Kenntnisse in der Anfragesprache von GIANT

  \end{itemize}

\section{Starten von GIANT}

Der Benutzer startet das Programm GIANT durch Ausführen der entsprechenden
Programmdatei. Hierbei hat er bezüglich eventueller Kommandozeilenparameter
die folgenden Optionen. 

  \subsection {Mögliche Parameter}
  \begin {enumerate}
    
    \item loadiml=''Pfad''\\
    Angabe des Pfades zu einer existierenden IML-Graph Datei.
    
    \item newprojectfile=''Pfad''\\
    Angabe des Pfades für die neu zu erstellenden Projektdatei 
    (darf nicht bereits vorhanden sein, sonst Fehlermeldung).\\
    Das Verzeichnis der Projektdatei ist automatisch Projektverzeichnis,
    falls schon vorhanden darf darin keine weitere Projektdatei liegen
    (sonst Fehlermeldung).\\
    Existiert das Projektverzeichnis noch nicht, so wird es ebenfalls angelegt.
    
    \item loadproject=''Pfad''\\
    Angabe des Pfades zu einer existierenden Projektdatei.
      
    \item executequery=''Anfrage''\\
    Eingabe eines Wortes der Anfragespreche als String.
     
    \item loadquery=''Pfad''\\
    Pfad zu einer Datei in der ein Wort der Anfragesprache gespeichert ist.
      
  \end {enumerate}
      
  \subsection {Mögliche Aufrufe}

  \begin {enumerate}
  
    \item Eingabe von: <giant>\\
    GIANT startet und ziegt das Main Window an. 
    Es ist noch kein Projekt geladen.
    
    \item Eingabe von: \\
    <giant -loadiml=''my\_iml\_file''\\
    -newprojectfile=''/home/projektverzeichnis/projektdatei''>\\
    Ein neues Projekt wird geöffnet und erhält den Namen der Projektdatei.
    \label{Kommandozeile_neus}

    \item Eingabe von: <giant -loadproject=''my\_project\_file''>\\
    Das angegebene Projekt wird geöffnet.
    \label{Kommandozeile_altes}
    
    \item
    Bei \ref{Kommandozeile_neus} und \ref{Kommandozeile_altes}
    kann zusätzlich entweder der Parameter\\
    <... -executequery''my\_query\_string''> oder der Parameter\\
    <... -loadquery''my\_query\_file''> eingegeben werden.\\
    In diesem Falle wird dann die übergebene Anfrage ausgeführt.
    Die Ergebnisse werden dem neuen oder dem bestehenden Projekt 
    hinzugefügt, sind aber noch nicht persistent gespeichert.


  \end {enumerate}

\section{Beenden von GIANT}
Giant kann durch Schließen des HAUPTFENSTERS oder durch Auswahl 
des entsprechenden Eintages im Hauptmenü oder durch ALT+F4 beendet werden.\\
Ablauf:

\begin {enumerate}
  \item Der Benutzer gibt das Kommando zum Beenden von GIANT.
  \item 
  Ist ein Projekt geöffnet, so fragt GIANT nach, ob das Projekt gespeichert
  werden soll. Falls der Benutzer das Speichern ablehnt, so gehen alle
  nicht gespeicherten Informationen verloren.\\
  Entscheidet er sich für speichern, so wird 
  die unter \ref{Alles Speichern} beschriebene Funktionaliät 
  ausgeführt.
 
    
  \item 
  Das System GIANT wird beendet.
    
\end {enumerate}
