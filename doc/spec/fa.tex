\section{Der Aktor \gq{Benutzer}} 
\index{Aktor} \index{Benutzer}
Aktor im Sinne der anschließend spezifizierten UseCases ist immer die 
Person, die den IML-Browser GIANT gerade bedient, also der 
\gq{Benutzer}. Neben dem \gq{Benutzer} sind keine weiteren Aktoren 
vorgesehen.\\


  \subsection{Anforderungen an den Aktor \gq{Benutzer}}
  Zur Bedienung von GIANT muss der Benutzer über 
  die folgenden Kenntnisse verfügen:

  \begin{enumerate}
    \item Erfahrung im Umgang mit graphischen Benutzeroberflächen und 
          dem entsprechenden Betriebssystem.
  
    \item Kenntnisse über Struktur und Aufbau des IML-Graphen. 
    
    \item Kenntnisse in der Skriptsprache GQSL von GIANT.

  \end{enumerate}

\section{Starten von GIANT} \index{Starten von GIANT}
\label{fa Starten von GIANT}

\index{Kommandozeile}
Der Benutzer startet das Programm GIANT durch Ausführen der entsprechenden
Programmdatei. Hierbei hat er bezüglich eventueller Kommandozeilenparameter
die folgenden Optionen. 

  \subsection {Mögliche Parameter} \index{Kommandozeilenparameter}

  GIANT kann folgendermaßen gestartet werden:\\\\
  \verb+giant [(-n file -i file|-l file) [-e file]|-h]+\\\\

  Hier werden die Parameter im einzelnen erklärt:

  \begin {enumerate}

  \item
    \verb+--newproject=Pfad+ oder \verb+-n Pfad+

    Angabe des Pfades für die neu zu erstellenden Projektdatei.  Die
    Datei darf nicht vorhanden sein. Das Verzeichnis der Projektdatei ist
    automatisch Projektverzeichnis und muss bereits existieren.

    \verb+--iml=Pfad+ oder \verb+-i Pfad+

    Angabe des Pfades zu einer existierenden IML-Graph Datei.
    
  \item 
    \verb+--loadproject=Pfad+ oder \verb+-l Pfad+

    Angabe des Pfades zu einer existierenden Projektdatei.
          
  \item
    \verb+--execute=Pfad+ oder \verb+-e Pfad+

    Pfad zu einer Datei in der ein GQSL Skript gespeichert ist.
  
  \item 
    \verb+--help+ oder \verb+-h+

    Zeigt eine Hilfe zu den möglichen Kommandozeilenparametern an.
      
  \end {enumerate}
      
  \subsection {Mögliche Aufrufe}

  Hier werden mögliche Parameterfolgen zum Start von GIANT via
  Kommandozeile erläutert.

  \begin {enumerate}
  
    \item Eingabe von: \verb+giant+\\
    GIANT startet und zeigt das Main Window an. 
    Es ist noch kein Projekt geladen.
    
    \item Eingabe von: \verb+giant --newproject=/home/projekt/sample.giant --iml=sample.iml+\\
    Ein neues Projekt wird geöffnet und erhält den Namen der Projektdatei.
    \label{Kommandozeile_neus}

    \item Eingabe von: \verb+giant --loadproject=sample.giant+\\
    Das angegebene Projekt wird geöffnet.
    \label{Kommandozeile_altes}
    
    \item
    Bei \ref{Kommandozeile_neus} und \ref{Kommandozeile_altes}
    kann zusätzlich der Parameter \verb+--execute=file.script+ 
    eingegeben werden.\\
    In diesem Falle wird das übergebene GQSL Skript ausgeführt.

  \end {enumerate}

\section{Beenden von GIANT} \index{Beenden von GIANT}
Ablauf:

\begin {enumerate}
  \item GIANT wählt \gq{Quit} im Hauptmenü 
    
  \item Ist ein Projekt geöffnet, so fragt GIANT nach, ob das Projekt
    gespeichert werden soll. Falls der Benutzer das Speichern ablehnt,
    so gehen alle nicht gespeicherten Informationen verloren.
    Entscheidet er sich für Speichern, so wird die unter \ref{Alles
      Speichern} beschriebene Funktionalität ausgeführt.
     
  \item GIANT wird beendet.
    
\end {enumerate}
